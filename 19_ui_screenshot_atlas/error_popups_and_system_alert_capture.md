# Error Popups And System Alert Capture

## Definition
A prompt atlas entry for staging error popups and system alert capture as a still image, collage panel, or screenshot-native environment.

## What Makes It Read Correctly
- 404s, broken embeds, warning dialogs, permissions, loading hangs, install bars, and browser nags
- crop logic matters as much as objects
- compression, aliasing, cursor traces, and text density help sell the capture
- the image should feel found, saved, reposted, or screen-grabbed rather than freshly rendered

## Framing Rules
- include the borders, rails, and interface furniture that define the screenshot type
- let some information be cut off at the edges
- include traces of navigation, scroll position, tabs, toolbars, or system chrome where appropriate
- treat the screenshot as an artifact with context, not a pure abstract composition

## Damage and Patina
- JPEG noise, scaling blur, RGB fringing, and old-screen glow are welcome
- repeated recapture can flatten contrast and thicken text halos
- timestamps, tiny icons, and unreadable microtext can function as texture

## Prompt Language
error popups and system alert capture, captured interface artifact, cropped screen ecology, reposted screenshot logic, archive screenshot patina, visible navigation residue
