# Dashboard Feed Capture

## Definition
A prompt atlas entry for staging dashboard feed capture as a still image, collage panel, or screenshot-native environment.

## What Makes It Read Correctly
- feed panels, reblog chains, notes, tags, reaction trails, comment drift, and stacked post modules
- crop logic matters as much as objects
- compression, aliasing, cursor traces, and text density help sell the capture
- the image should feel found, saved, reposted, or screen-grabbed rather than freshly rendered

## Framing Rules
- include the borders, rails, and interface furniture that define the screenshot type
- let some information be cut off at the edges
- include traces of navigation, scroll position, tabs, toolbars, or system chrome where appropriate
- treat the screenshot as an artifact with context, not a pure abstract composition

## Damage and Patina
- JPEG noise, scaling blur, RGB fringing, and old-screen glow are welcome
- repeated recapture can flatten contrast and thicken text halos
- timestamps, tiny icons, and unreadable microtext can function as texture

## Prompt Language
dashboard feed capture, captured interface artifact, cropped screen ecology, reposted screenshot logic, archive screenshot patina, visible navigation residue

## Atlas Seeds
1. a dashboard feed capture with cut-off text and browser chrome still visible
2. a reposted low-res dashboard feed capture floating inside a larger collage of internet debris
3. a highly legible composition built from the anatomy of dashboard feed capture rather than generic app design
4. a nostalgic interface scene whose realism comes from crop awkwardness and screenshot residue
5. a damaged or recaptured version of dashboard feed capture with subtle artifact build-up and timestamp fossils
