# Full Page Scroll Capture

## Definition
A prompt atlas entry for staging full page scroll capture as a still image, collage panel, or screenshot-native environment.

## What Makes It Read Correctly
- stitched page screenshots with seam ghosts, repeated headers, drifting sidebars, partial scrollbars, and vertical narrative accumulation
- crop logic matters as much as objects
- compression, aliasing, cursor traces, and text density help sell the capture
- the image should feel found, saved, reposted, or screen-grabbed rather than freshly rendered

## Framing Rules
- include the borders, rails, and interface furniture that define the screenshot type
- let some information be cut off at the edges
- include traces of navigation, scroll position, tabs, toolbars, or system chrome where appropriate
- treat the screenshot as an artifact with context, not a pure abstract composition

## Damage and Patina
- JPEG noise, scaling blur, RGB fringing, and old-screen glow are welcome
- repeated recapture can flatten contrast and thicken text halos
- timestamps, tiny icons, and unreadable microtext can function as texture

## Prompt Language
full page scroll capture, captured interface artifact, cropped screen ecology, reposted screenshot logic, archive screenshot patina, visible navigation residue

## Atlas Seeds
1. a full page scroll capture with cut-off text and browser chrome still visible
2. a reposted low-res full page scroll capture floating inside a larger collage of internet debris
3. a highly legible composition built from the anatomy of full page scroll capture rather than generic app design
4. a nostalgic interface scene whose realism comes from crop awkwardness and screenshot residue
5. a damaged or recaptured version of full page scroll capture with subtle artifact build-up and timestamp fossils
